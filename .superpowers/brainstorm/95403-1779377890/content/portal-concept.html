Client Portal — How should clients access it?

Each client gets a view of their process(es). The question is: how do they log in and what do they see?

A

Shareable Link (no login)

Each process gets a unique, secret URL. Client bookmarks it, opens any time — no username/password. Like a Google Analytics "view-only" share link. Best for clients who just want to check in without friction.

Pros

  • Zero friction for client
  • No account management
  • Easy to revoke (rotate the token)

Cons

  • Link can be forwarded/leaked
  • No audit trail of who viewed
B

Client Login Portal

Each client company gets a login. One login can cover multiple processes under that client. Supports multiple client-side users (e.g., client's ops head + client's QA manager each have accounts).

Pros

  • Audit trail — who logged in when
  • Multiple contacts per client
  • Can send email alerts to client users

Cons

  • Client must manage a password
  • More setup per client
C

Both — Link for quick view, Login for full access

Public shareable link shows summary KPIs only (read-only snapshot). Full portal login unlocks action plans, governance checklists, trend history, and downloadable reports.

Pros

  • Covers all client types
  • Tiered access makes sensitive data safer

Cons

  • More complexity to build
  • Need to define what's in each tier
